3. untuk menjalankan atau menghentikan daemon samba, smbd dan nmbd dapat diatur secara otomatis melalui skrip


/etc/rc.d/init.d/smbUntuk menjalankan atau menghentikan daemon samba, smbd dan nmbd dapat diatur secara otomatis dengan menambahkan skrip berikut di /etc/rc.local:

menjalankan:
/etc/init.d/samba start
/etc/init.d/smbd start
/etc/init.d/nmbd start

menghentikan:
/etc/init.d/samba stop
/etc/init.d/smbd stop
/etc/init.d/nmbd stop

Jawaban:

Penjelasan:

Muncul pesan error "failed to start daemon: pid file found, ensure docker is not running or delete /var/run/docker.pid" menunjukkan bahwa terdapat proses Docker yang sedang berjalan atau terdapat berkas pid (process identification) Docker yang masih ada di direktori /var/run. Untuk mengatasi masalah ini, Anda dapat mencoba salah satu dari tindakan berikut:

Matikan proses Docker yang sedang berjalan dengan perintah "sudo docker stop [container_id]". Kemudian coba jalankan perintah "sudo service docker start" dan "sudo dockerd" lagi.

Hapus berkas pid Docker yang masih ada di direktori /var/run dengan perintah "sudo rm /var/run/docker.pid". Kemudian coba jalankan perintah "sudo service docker start" dan "sudo dockerd" lagi.

Restart sistem Anda dan coba jalankan perintah "sudo service docker start" dan "sudo dockerd" lagi setelah sistem booting kembali.

Jika masalah masih belum teratasi, Anda dapat mencoba menggunakan perintah "sudo dockerd -D" untuk menjalankan daemon Docker dengan mode debugging dan memperoleh informasi lebih lanjut tentang masalah yang terjadi.
